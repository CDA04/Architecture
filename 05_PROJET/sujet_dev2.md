
## Développer une application sécurisée

| **Activité type**                            | **Compétences professionnelles**                                         | **Savoirs-faire**                                                                                                   |
|----------------------------------------------|---------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------|
| **1 - Développer une application sécurisée** | **1.1 - Installer et configurer son environnement de travail**            | - Les outils de développement nécessaires sont installés.<br>- Les outils de gestion des versions et de collaboration sont installés.<br>- Les conteneurs implémentent les services requis.<br>- La documentation technique de l’environnement de travail est comprise en français ou anglais. |
|                                              | **1.2 - Développer des interfaces utilisateur**                           | - L’interface est conforme au dossier de conception.<br>- L’interface s’adapte au type d’utilisation de l’application, notamment à la taille, au type et à la disposition du support.<br>- La charte graphique est respectée.<br>- La réglementation en vigueur est respectée.<br>- Le code est documenté.<br>- Les tests unitaires ont été réalisés pour les composants concernés.<br>- Le jeu d’essai fonctionnel est complet.<br>- Les tests de sécurité sont réalisés.<br>- La documentation technique des interfaces utilisateur est comprise en français ou anglais.<br>- La démarche structurée de résolution de problème est adaptée en cas de dysfonctionnement.<br>- Le système de veille permet de suivre les évolutions technologiques et les problématiques de sécurité en lien avec les interfaces utilisateur. |
|                                              | **1.3 - Développer des composants métier**                                | - Les bonnes pratiques de la programmation orientée objet (POO) sont respectées.<br>- Les composants métier sont sécurisés.<br>- Les règles de nommage sont conformes aux normes de qualité de l’entreprise.<br>- Le code source est documenté.<br>- Les traitements répondent aux fonctionnalités décrites dans le dossier de conception.<br>- Les tests unitaires sont réalisés.<br>- Les tests de sécurité sont réalisés.<br>- La démarche structurée de résolution de problème est adaptée en cas de dysfonctionnement.<br>- Le système de veille permet de suivre les évolutions technologiques et les problématiques de sécurité en lien avec les composants métier d’une application. |
|                                              | **1.4 - Contribuer à la gestion d’un projet informatique**                | - Les tâches de conception et de développement sont planifiées en fonction du délai défini.<br>- Le suivi des tâches est mis en rapprochement avec la planification, les éventuels retards sont identifiés et les acteurs concernés sont alertés.<br>- Les procédures qualité sont mises en œuvre.<br>- L’environnement de développement défini est en adéquation avec l’architecture du projet.<br>- Les outils collaboratifs sont choisis en fonction de la méthode de développement.<br>- Les comptes rendus de réunion sont structurés, rédigés dans un style adapté, dans le respect des règles orthographiques et grammaticales, et contiennent les informations nécessaires. |

## Concevoir et développer une application sécurisée organisée en couches

| **Activité type**                                      | **Compétences professionnelles**                                               | **Savoirs-faire**                                                                                                   |
|--------------------------------------------------------|---------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------|
| **2 - Concevoir et développer une application sécurisée organisée en couches** | **2.5 - Analyser les besoins et maquetter une application**                     | - Les besoins recensés couvrent l'ensemble des exigences utilisateur exprimées dans le cahier des charges.<br>- Les maquettes sont réalisées conformément au cahier des charges en langue française ou anglaise.<br>- L’enchaînement des maquettes est formalisé par un schéma.<br>- Le dossier de conception est structuré, en conformité avec la démarche de conception. |
|                                                        | **2.6 - Définir l’architecture logicielle d’une application**                   | - Définir l’architecture logicielle en identifiant les Framework et ORM à utiliser.<br>- Adapter l’architecture logicielle aux besoins des utilisateurs et à la stratégie de sécurité selon les recommandations de l’ANSSI.<br>- Utiliser les patrons de conception (design patterns) et les patrons de sécurité (Security patterns). |
|                                                        | **2.7 - Concevoir et mettre en place une base de données relationnelle**         | - Le schéma conceptuel respecte les règles du relationnel.<br>- Le schéma physique est conforme aux besoins exprimés dans le cahier des charges.<br>- Les règles de nommage ont été respectées.<br>- L’intégrité, la sécurité et la confidentialité des données sont assurées.<br>- La base de données de test est créée avec un jeu d’essai complet et peut être restaurée en cas d’incident.<br>- La documentation technique des bases de données est comprise, en langue française ou anglaise. |
|                                                        | **2.8 - Développer des composants d’accès aux données SQL et NoSQL**            | - Les traitements relatifs aux manipulations des données répondent aux fonctionnalités décrites dans le dossier de conception.<br>- Les cas d'exception sont pris en compte.<br>- L'intégrité et la confidentialité des données sont maintenues.<br>- Les conflits d'accès aux données sont gérés.<br>- Toutes les entrées sont contrôlées et validées dans les composants serveurs sécurisés.<br>- Les tests unitaires et de sécurité sont associés à chaque composant.<br>- La démarche structurée de résolution de problème est adaptée en cas de dysfonctionnement.<br>- Le système de veille permet de suivre les évolutions technologiques et les problématiques de sécurité liées aux bases de données SQL et NoSQL. |


## Préparer le déploiement d’une application sécurisée** | **3.9 - Préparer et exécuter les plans de tests d’une application

| **Activité type**                                      | **Compétences professionnelles**                                               | **Savoirs-faire**                                                                                                   |
|--------------------------------------------------------|---------------------------------------------------------------------------------|---------------------------------------------------------------------------------------------------------------------|
| **3 - Préparer le déploiement d’une application sécurisée** | **3.9 - Préparer et exécuter les plans de tests d’une application**              | - Le plan de tests couvre l'ensemble des fonctionnalités retenues pour l’application.<br>- Un environnement de tests est créé.<br>- L’intégralité des tests exécutés sont conformes au plan de tests défini.<br>- Les résultats obtenus sont cohérents avec les résultats attendus.<br>- Le plan de tests tient compte des évolutions technologiques et des problèmes de sécurité liés aux tests logiciels. |
|                                                        | **3.10 - Préparer et documenter le déploiement d’une application**               | - La procédure de déploiement est rédigée.<br>- Les scripts de déploiement sont écrits et documentés.<br>- Les environnements de tests sont définis et la procédure d’exécution des tests d’intégration, système et d’acceptation client est rédigée.<br>- Le système de veille permet de suivre les évolutions technologiques et les problématiques de sécurité liées au déploiement d’une application. |
|                                                        | **3.11 - Contribuer à la mise en production dans une démarche DevOps**            | - Les outils de qualité de code sont utilisés.<br>- Les outils d’automatisation de tests sont utilisés.<br>- Les scripts d’intégration continue s’exécutent sans erreur.<br>- Le serveur d’automatisation est paramétré pour les livrables et les tests.<br>- Les rapports de l’Intégration Continue sont interprétés.<br>- La documentation technique des différents outils est comprise, en langue française ou anglaise.<br>- La démarche structurée de résolution de problème est adaptée en cas de dysfonctionnement.<br>- Le système de veille permet de suivre les évolutions technologiques et les problématiques de sécurité liées à la démarche DevOps. |
